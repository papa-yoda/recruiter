# Reference

## Skills

### `/scan-jobs`

Fetches LinkedIn job descriptions and assigns each to a resume category.

| Argument | Effect |
|----------|--------|
| *(none)* | Process only new (unprocessed) URLs |
| `--force` | Re-process all URLs |

On the first run with no existing categories, proposes 2-5 groupings based on patterns across your JDs. On subsequent runs, matches new jobs to existing categories and proposes new ones only if needed. Always asks for approval before saving category changes.

### `/generate-resumes`

Creates a tailored resume markdown file for each category.

| Argument | Effect |
|----------|--------|
| *(none)* | Generate only for categories with new (unprocessed) jobs |
| `--force` | Regenerate all resumes |
| `--category <name>` | Generate for a specific category only |

### `/build-pdfs`

Runs `build-resumes.sh` to convert resume markdowns to styled PDFs.

| Argument | Effect |
|----------|--------|
| `--category <name>` | Build only a specific category |

Requires `pandoc` and `weasyprint` installed locally.

### `/sync-notion`

Creates or updates Notion database entries for processed jobs.

| Argument | Effect |
|----------|--------|
| *(none)* | Sync only jobs not yet in Notion |
| `--force` | Re-sync all jobs (won't overwrite Status) |

### `/recruiter`

Runs the full pipeline: scan -> generate -> build -> sync.

| Argument | Effect |
|----------|--------|
| `--force` | Re-process everything from scratch |
| `--skip-pdf` | Skip PDF generation step |
| `--skip-notion` | Skip Notion sync step |

## Configuration

### `config.yaml`

Comments in the file explain each field. The key sections:

- **candidate** — your name, location, email, LinkedIn (used in resume headers and PDF filenames)
- **notion** — database ID and data source ID for Notion tracking. Leave `database_id` blank to disable.
- **paths** — file paths for inputs/outputs. Defaults work unless you rename files.

### `categories.yaml`

Auto-generated by `/scan-jobs`, but you can edit it manually. Each category has:

- **name** — display label (e.g., "Frontend Engineering")
- **description** — one-line summary to help match future jobs
- **signals** — JD keywords that indicate this category
- **file_pattern** — used in resume filenames; text in parentheses becomes the PDF subdirectory

To rename a category, update all four fields and run `/generate-resumes --force`.

## Notion Setup

1. Create a new full-page Notion database with these properties:

| Property | Type |
|----------|------|
| Job Title | title |
| Company | text |
| Job Link | url |
| Status | select (Not Applied, Applied, Interview, Offer, Rejected) |
| Resume Version | select |
| Category | select |
| Salary Min | number |
| Salary Max | number |
| Notes | rich text |
| Year | number |

2. Copy the database ID from the URL: `https://notion.so/workspace/<database_id>?v=...`
3. Paste it into `config.yaml` under `notion.database_id`
4. Ensure the Notion MCP integration in Claude Code has access to the database

## Customization

### PDF Styling

Edit `Generated Resumes/resume-style.css`. Defaults: US Letter, 0.5" margins, 10pt system sans-serif.

### Resume Generation Rules

Edit `.claude/skills/generate-resumes/SKILL.md` to change bullet count per role, summary length, formatting, or prioritization.

### Non-LinkedIn Sources

To add jobs from other sources, edit `.processed-jobs.yaml` directly — add an entry with `jd_summary` filled in, then run `/generate-resumes`.
