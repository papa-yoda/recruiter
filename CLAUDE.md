# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Repository Purpose

A shareable template for AI-powered job searching. Users add their work experience and LinkedIn job URLs, and Claude Code skills handle: fetching job descriptions, discovering resume categories, generating tailored resumes, building PDFs, and syncing to Notion for application tracking.

This template is **role-agnostic** — it works for any job type, industry, or career level. Categories are discovered dynamically from job descriptions, not predefined.

## Key Files

### Configuration
- **config.yaml** — Candidate info (name, email, location, LinkedIn), Notion database IDs, file paths. Edit this first.
- **categories.yaml** — Auto-generated by `/scan-jobs`. Stores discovered resume categories and their signals. Can also be edited manually.

### Input
- **work-experience.md** — Complete career history in stream-of-consciousness format. Intentionally verbose — never clean up. Organized as Company > Title > Projects with tags for filtering.
- **LinkedIn Jobs.md** — LinkedIn job URLs, one per line. Lines starting with `#` are comments.

### Output
- **Generated Resumes/** — Resume markdown files and PDFs, organized by category subdirectories.

### Pipeline State
- **.processed-jobs.yaml** — Tracks which URLs have been processed, their categories, and pipeline status. Auto-managed by skills. Gitignored.

## Skills

| Skill | Purpose | Key Arguments |
|-------|---------|---------------|
| `/scan-jobs` | Fetch JDs from LinkedIn, discover/assign categories | `--force` (re-process all) |
| `/generate-resumes` | Create tailored resume MDs per category | `--force`, `--category <name>` |
| `/build-pdfs` | Convert MDs to PDFs via pandoc + weasyprint | `--category <name>` |
| `/sync-notion` | Create/update Notion database entries | `--force` (re-sync all) |
| `/recruiter` | Run full pipeline (all four steps) | `--force`, `--skip-pdf`, `--skip-notion` |

## Work Experience Format

```
# Company Name
Brief company description

## Job Title
Role description
**Tags:** #relevant #tags

### Date
Start - End

### Projects

#### Project Name
**Tags:** #project #tags
- Bullet points with quantified results
```

**Tags** enable filtering relevant experience. Use any tags meaningful to your field.

## Resume Generation Rules

When generating resumes:
1. Read `work-experience.md` as the sole source of truth for career history
2. Read `categories.yaml` for the target category's signals and description
3. Read `.processed-jobs.yaml` for JD summaries to inform keyword targeting
4. Key constraints:
   - 3-5 bullet points max per role, no subsections
   - Quantify everything (numbers, percentages, dollar amounts)
   - Match JD language exactly (mirror the terminology from job descriptions)
   - Prioritize: relevance to JDs > quantified impact > recency > breadth
   - Bold and bullets only — no italics, tables, or complex layouts (ATS-friendly)
   - **Never fabricate** — only use information present in `work-experience.md`

## File Naming Convention

Resume files: `{Candidate Name} - {file_pattern}.md`

The `file_pattern` comes from `categories.yaml` and always includes parentheses: `Resume (Category Name)`. The build script extracts the parenthesized text as the PDF subdirectory name.

Examples:
- `Jane Doe - Resume (Frontend Engineering).md` → `Frontend Engineering/Jane Doe Resume.pdf`
- `Jane Doe - Resume (Full-Stack Product).md` → `Full-Stack Product/Jane Doe Resume.pdf`

## PDF Build Pipeline

`Generated Resumes/build-resumes.sh` converts markdown resumes to PDFs:
- Requires `pandoc` and `weasyprint`
- Reads candidate name from `config.yaml` for output filename
- Uses `Generated Resumes/resume-style.css` for styling
- Output: `Generated Resumes/<Category>/Candidate Name Resume.pdf`

```bash
cd "Generated Resumes" && ./build-resumes.sh
```

## Notion Integration (Optional)

Uses Notion MCP tools to manage a Job Applications database. Configure in `config.yaml`:
- Set `notion.database_id` to your database ID
- Set `notion.data_source_id` for MCP access
- Database properties: Job Title, Company, Job Link, Status, Resume Version, Category, Salary Min/Max, Notes, Year

Leave `notion.database_id` blank to skip Notion integration entirely.

## Category Discovery

Categories are NOT predefined. On first run, `/scan-jobs` analyzes all job descriptions and proposes natural groupings based on:
- Shared technology stacks
- Domain keywords
- Role responsibilities
- Industry vertical

Categories are saved in `categories.yaml` and reused for future jobs. New categories can be proposed when jobs don't fit existing ones — the user always approves before changes are saved.
